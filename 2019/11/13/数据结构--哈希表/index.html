数据结构--集合

Posted on 2019-11-13 | In 数据结构

1. 哈希表

  • 基于数组实现。
  • 数组的插入删除效率低，而哈希表高
  • 查找的效率：如果用数组索引查找效率很高，如果用数组内容查，效率很低。而哈希表高
  • 特点：哈希表中的数据没有顺序
  • 特点：哈希表中的key不允许重复

2. 哈希表案例

  1. 存储公司员工信息。使用数组和链表都不合适。使用哈希函数将名字转为下标值。
  2. 联系人和电话号码存储。使用哈希函数将名字转为下标值。
  3. 50000个单词的存储。

3. 概念

  • 哈希化：把大的数字转为数值范围内下标的过程。
  • 哈希函数：通常将单词转为大数字，大数字进行哈希化的代码实现放在一个函数中，这个函数就叫做哈希函数。
  • 哈希表：最终将数据插入到的这个数组，对整个结构的封装，称之为哈希表。

4. 解决冲突方案

  • 链地址法(推荐使用)

  • 开放地址法
    • 线性探测：x+1
    • 二次探测: x+1^2,x+2^2
    • 再哈希化: step = constant - (key % constant)

5. 优秀的哈希函数

  • 均匀分布：常量的地方用质数
  • 快速计算：霍纳法则（提取公因子）

6. java中的HashMap

index = HashCode(key) & (Length - 1)
