数据结构--红黑树

Posted on 2019-11-17 | In 数据结构

1. 红黑树规则

  1. 结点是红色或者黑色
  2. 根节点是黑色
  3. 每个叶子结点都是黑色的空结点（NIL结点）
  4. 每个红色结点的两个子节点都是黑色。（从每个叶子到根的所有路径上，不能有两个连续的红色结点）
  5. 从任意一个结点到其他每个叶子节点的所有路径都包含相同数目的黑色结点

2. 红黑树的相对平衡

  • 从根到叶子的最长路径，不会超过最短可能路径的两倍
  • 结果基本平衡

3. 变色，左旋转，右旋转

插入的结点默认是红色。

4. 变换规则

4.1 情况1 和 情况2

情况1：

  • 新节点N位于根上，没有父节点

操作：

  • 这时候把红色改为黑色即可

情况2：

  • 新节点的父节点P是黑色

操作：

  • 直接插入红色的新节点N
4.2 情况3

情况3：

  • P(父节点)为红色，U(叔叔结点)也为红色
  • G(祖父结点)为黑色

操作：

  • 变成 -> 父黑叔黑祖红
4.3 情况4

情况4：

  • 父红树黑祖黑，N是左儿子

操作：

  • -> 父黑
  • -> 祖红
  • -> 右旋转
4.4 情况5

情况5：

  • 父红叔黑祖黑，N是右儿子

操作：

  • -> 以P为根进行左旋转
  • 将P作为新插入的红色结点考虑即可
    • 自己变成黑色
    • 祖变成红色
    • 以祖为根，进行右旋转
