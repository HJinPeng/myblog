css-overflow

Posted on 2019-03-21 | In css

1. overflow: hidden

遇到的情况：section设置了背景颜色但高度为auto，section中三个div都设置了float，背景颜色不显示

原因：所有div设置了float，导致section无法根据子元素撑起高度

解决方法：在父元素section中设置清除浮动section{overflow:hidden;} ,这样浮动的子元素又可以撑起高度了

ps：若父元素section设置了高度值，则浮动的元素超出范围的部分会被隐藏

2. overflow: auto

超出边界的值不显示，但显示默认样式滚动条

git--分支管理

Posted on 2019-03-20 | In git

1. 创建和合并分支

创建分支：git branch <name>

切换分支：git checkout <name>

创建和切换分支：git checkout -b <name>

查看分支：git branch

######例如：

git checkout -b dev表示创建dev分支并切换到dev分支

当前分支为master，git meger dev表示将分支dev合并到当前的master分支上

git--标签管理

Posted on 2019-03-20 | In git

1. 创建标签

  1. git tag <tagName> 新建标签，默认标签打在最新提交的commit上
  2. git tag <tagName> <commit id> 可以在历史提交中的某一个commit上打标签
  3. git tag可以查看存在的标签
  4. git tag -a <tagName> -m "说明文字" 可以创建带有说明的标签
  5. git show <tagName> 可以看到标签\的说明

2. 操作标签

  1. git tag -d <tagName> 删除标签
  2. git push origin <tagName> 将标签推送到远程库
  3. git push origin --tags 一次性推送全部尚未推送到远程的本地标签
  4. git tag -d <tagName> git push origin :refs/tags/<tagName> 先删除本地标签，再删除远程标签

git--远程仓库

Posted on 2019-03-18 | In git

1. 关联远程库

git remote add origin https://github.com/账号名/仓库名

2. 关联后第一次推送master分支

git push -u origin master

3. 本地提交后，再推送到远程

git push origin master

4. 克隆仓库

git clone https://github.com/账号名/仓库名
