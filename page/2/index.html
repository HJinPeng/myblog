css--浏览器兼容

Posted on 2019-05-21 | In css

1. 图片

img的默认属性inline-block会使得下面有个间隙
解决方法：
将img设置为block

2. 双倍边距

当ie6及更低版本浏览器在解析浮动元素的时候，会错误的把浮动边界加倍显示
解决方法：
给浮动元素添加声明: display: inline;

3. 默认高度（ie6）

ie6及其以下版本中，部分块元素拥有默认高度（18px高度）
解决方案：

  1. 给元素添加声明：font-size:0
  2. 给元素添加声明：overflow:hidden

4. 表单元素的行高不一致

在ie6下表单元素的高度不一致，如按钮文字的高度和input中文字的高度不一致
解决方案：
给表单元素添加float:left

5. 按钮元素默认大小不一样

解决方案：

  1. 使用a标签制作按钮
  2. 在input上写按钮的样式，要把input的边框去掉
  3. 如果按钮是一张图片，直接把图片作为按钮的背景图

6. 百分比bug

在ie6及以下浏览器中解析百分比时，会按四舍五入计算，导致50%+50%>100%
解决方法：

  1. 给右边的浮动元素添加声明：clear:right

7. 鼠标cursor

cursor: hand只有ie浏览器能识别，cursor: pointer;在ie6以上能识别
解决方案：
使用兼容更多浏览器的属性

  1. 下划线过滤器

css--盒模型

Posted on 2019-05-21 | In css

盒模型

标准盒子是：content(width和height)+padding+border+margin

0. 清除margin和padding的默认值

像h标签就有默认的margin值，为了不影响后面的计算，首先要清除所有元素的margin和padding值

1. width和height

宽度和高度其实是content的宽高，content外面是padding，border，margin

2. padding 和 border 撑大容器

添加 padding 和 border 都会撑大原来的块的大小，为了保证原来的块大小不变，所以要把width（其实就是content的宽和高）减掉padding(border)的左右宽度之和
把height减掉padding(border)的上下高度之和

3. margin

  • 相邻兄弟在垂直方向上的margin值会出现融合的情况，融合后会取两个元素中较大的值作为融合后的值

  • 当父元素和第一个子元素都设置了margin-top时，第一个子元素的margin-top值会传递到父元素，然后按最大的值进行设置，解决方法是在父元素上添加overflow:hidden;

HTML--版本声明

Posted on 2019-05-21 | In HTML

Html 版本声明

一、 html 5

<!DOCTYPE html>

二、 html 4.0.1

1. html 4.0.1 Strict

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd">
该DTD包括html的元素和属性，但不包括展示性的和被弃用的元素。不允许框架集

2. html 4.0.1 Transitional

该DTD包括html的元素和属性，包括展示性和被弃用的元素，但不允许框架集

3. html 4.0.1 Frameset

该DTD包括html的元素和属性，包括展示性的和被弃用的元素，也包括框架集

三、 XHTML 1.0

XHTML 1.0 和 html 4.0.1 一样有三种模式，但必须以格式正确的XML来编写标记

1. XHTML 1.0 Strict

2. XHTML 1.0 Transitional

3. XHTML 1.0 Frameset

四、 XHTML 1.1

该DTD等同于XHTML1.0 Strict ，但允许添加模型

git--忽略文件.gitignore

Posted on 2019-05-06 | In git

.gitignore内容

#后的内容为注释，会被忽略
.txt　　 #会忽略所有.txt结尾的文件
!test.txt　　 #test.txt除外，即不被忽略
/todo　　 #仅忽略项目根目录下的 TODO 文件，不包括 subdir/todo
build/ 　　 # 忽略 build/ 目录下的所有文件
doc/.txt 　　# 会忽略 doc/notes.txt 但不包括 doc/server/arch.txt

.gitignore不生效的解决方法

把某些目录或文件加入忽略规则，按照上述方法定义后发现并未生效，原因是.gitignore只能忽略那些原来没有被追踪的文件，如果某些文件已经被纳入了版本管理中，则修改.gitignore是无效的。那么解决方法就是先把本地缓存删除（改变成未被追踪状态），然后再提交：
git rm -r --cached .
git add .
git commit -m "message"
