css--弹性盒子

Posted on 2019-05-21 | In css

1. 父元素的属性

flex 和 inline-flex

flex-direction

主轴方向

  1. row 默认，从左到右
  2. row-reverse 横向从右到左
  3. column 主轴变成从上到下
  4. column-reserse 主轴从下到上

flex-wrap

  1. nowrap 默认值，当子元素溢出父元素时不换行，会对子元素在宽度上进行删减，使得全部排进一行
  2. wrap 当子元素溢出父元素时换行
  3. wrap-reverse 反转wrap排列

justify-content

设置弹性盒子子元素在主轴方向上的对齐方式

  1. flex-start 子元素向行起始位置对齐
  2. flex-end 子元素向行结束位置对齐
  3. center 子元素向中间位置对齐
  4. space-between 子元素会平均分配到行里
  5. space-around 子元素会平局分布在行里且两端保留子元素和子元素之间间距大小的一半

align-items

设置子元素在测轴方向上的对齐方式

  1. flex-start
  2. flex-end
  3. center
  4. baseline 将文字对齐
  5. stretch 如果为设置宽高，那么将元素默认拉伸为父元素高度

align-content

用于修改flex-wrap属性的行为，类似于justify-content，但它不是对子元素的对齐，而是对各个行的对齐
当flex-wrap: wrap时，产生了几行，使用下面的属性可以设置行之间的间距即测轴方向的对齐方式

  1. flex-start
  2. flex-end
  3. center
  4. space-between
  5. space-around
  6. stretch 默认值

css--浏览器兼容

Posted on 2019-05-21 | In css

1. 图片

img的默认属性inline-block会使得下面有个间隙
解决方法：
将img设置为block

2. 双倍边距

当ie6及更低版本浏览器在解析浮动元素的时候，会错误的把浮动边界加倍显示
解决方法：
给浮动元素添加声明: display: inline;

3. 默认高度（ie6）

ie6及其以下版本中，部分块元素拥有默认高度（18px高度）
解决方案：

  1. 给元素添加声明：font-size:0
  2. 给元素添加声明：overflow:hidden

4. 表单元素的行高不一致

在ie6下表单元素的高度不一致，如按钮文字的高度和input中文字的高度不一致
解决方案：
给表单元素添加float:left

5. 按钮元素默认大小不一样

解决方案：

  1. 使用a标签制作按钮
  2. 在input上写按钮的样式，要把input的边框去掉
  3. 如果按钮是一张图片，直接把图片作为按钮的背景图

6. 百分比bug

在ie6及以下浏览器中解析百分比时，会按四舍五入计算，导致50%+50%>100%
解决方法：

  1. 给右边的浮动元素添加声明：clear:right

7. 鼠标cursor

cursor: hand只有ie浏览器能识别，cursor: pointer;在ie6以上能识别
解决方案：
使用兼容更多浏览器的属性

9. 过滤器

  1. 下划线过滤器
    当在一个属性前加了下划线，符合标准的浏览器不能识别带下划线的属性，于是忽略了这个属性，但是ie6及更低版本的浏览器会继续识别并解析

  2. 属性过滤器
    当一个属性加了    号，只能被ie7浏览器识别，其他浏览器会忽略
