2. TCP和UDF的区别

相同点：
TCP和UDP都是传输层协议。

  • TCP是面向连接，可靠的字节流服务。能提供超时重发，丢弃重复数据，检验数据，流量控制等功能，保证数据能从一端到另一端。
  • UDP是面向数据报的不可靠运输层协议。它只是把数据报发送出去，但是不能保证它们能到达目的地，由于UDP在传输数据报前不用再客户端和服务器端之间建立一个连接，且没有超时重发等机制，所以传输很快。

不同点：

  1. 报头不同
  2. 特点不同
  3. 协议不同

3. TCP三次握手、四次挥手

  1. 三次握手过程


第一次握手： 客户端发送SYN=1（请求连接）给服务端。

第二次握手： 服务端收到建立连接请求，也发送SYN=1（请求连接）和ACK=1（确认有效）给客户端。【客户端就知道了服务端能收到消息】

第三次握手： 客户端收到服务端发来的SYN=1和ACK=1，知道了服务端收到了请求，于是发送了ACK=1给服务端，告诉服务端也收到了。【服务端就知道了客户端能收到消息】

  1. 四次挥手过程

第一次握手： 当客户端完成数据传输后，将FIN=1，提出停止TCP连接的请求。

第二次握手： 服务端收到FIN=1，发送ACK=1，确认这方向上的TCP连接将关闭

第三次握手： 服务器端再发送关闭请求，FIN=1

第四次握手： 客户端发送ACK=1到服务端，双方关闭。

4. 为什么连接的时候是三次握手，关闭的时候却是四次握手？

因为连接时，服务端收到客户端的SYN连接请求报文后，可以直接发送SYN同步和ACK应答。但是关闭连接时，当服务端收到FIN结束报文时，很可能一些报文还没发送完，所以先回复了一个ACK应答报文，告诉浏览器：我已经收到FIN报文。只有服务端的所有报文都发送完毕后，服务端才会再发送一个FIN报文给客户端。故需要四次握手（挥手）

css--弹性盒子

Posted on 2019-05-21 | In css

1. 父元素的属性

flex 和 inline-flex

flex-direction

主轴方向

  1. row 默认，从左到右
  2. row-reverse 横向从右到左
  3. column 主轴变成从上到下
  4. column-reserse 主轴从下到上

flex-wrap

  1. nowrap 默认值，当子元素溢出父元素时不换行，会对子元素在宽度上进行删减，使得全部排进一行
  2. wrap 当子元素溢出父元素时换行
  3. wrap-reverse 反转wrap排列

justify-content

设置弹性盒子子元素在主轴方向上的对齐方式

  1. flex-start 子元素向行起始位置对齐
  2. flex-end 子元素向行结束位置对齐
  3. center 子元素向中间位置对齐
  4. space-between 子元素会平均分配到行里
  5. space-around 子元素会平局分布在行里且两端保留子元素和子元素之间间距大小的一半

align-items

设置子元素在测轴方向上的对齐方式

  1. flex-start
  2. flex-end
  3. center
  4. baseline 将文字对齐
  5. stretch 如果为设置宽高，那么将元素默认拉伸为父元素高度

align-content

用于修改flex-wrap属性的行为，类似于justify-content，但它不是对子元素的对齐，而是对各个行的对齐
当flex-wrap: wrap时，产生了几行，使用下面的属性可以设置行之间的间距即测轴方向的对齐方式

  1. flex-start
  2. flex-end
  3. center
  4. space-between
  5. space-around
  6. stretch 默认值
